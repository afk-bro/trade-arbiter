# Project Updates

> Append-only log of what shipped, what's in flight, and what's coming next.

This is the **narrative** companion to the roadmap. Where `roadmap.md` is a status table, this directory holds short prose write-ups: what landed this week, what got cut, what surprised us, what an outsider catching up on the project should know.

## Cadence

Write an update when one of these happens:

- A plan ships (or slips by more than a few days)
- A scope decision changes the roadmap
- An incident worth remembering occurred
- Enough time has passed that the project state has drifted from the last update

There is no required frequency. Better to have three substantive updates a month than twelve empty ones.

## Format

Use [`_template.md`](_template.md). Filename: `YYYY-MM-DD-short-slug.md`. Updates are append-only — once written, treat them as the historical record. If something turns out to be wrong, write a follow-up update rather than editing history.
